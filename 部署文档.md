# VCD管理系统 - 完整部署文档

## 一、环境要求

| 环境 | 版本要求 |
|------|---------|
| JDK | 17 |
| Maven | 3.6+ |
| SQL Server | 2019 / 2022 |
| Node.js | 18+ |
| npm | 9+ |

---

## 二、数据库部署步骤

### 1. 确认 SQL Server 已启动
- 打开「SQL Server 配置管理器」，确认 SQL Server 服务正在运行
- 确认 TCP/IP 协议已启用，端口为 **1433**

### 2. 确认 sa 账户可用
- 打开 SSMS（SQL Server Management Studio）
- 使用 `sa` / `123456` 登录（如密码不同，需修改 `application.properties`）

### 3. 执行数据库脚本
打开 SSMS，新建查询窗口，执行以下步骤：

**第一步：创建数据库**
```sql
CREATE DATABASE vcddb;
```

**第二步：执行建表和初始数据脚本**
- 打开文件：项目根目录下的 `script.sql`
- 全选内容，在 SSMS 中执行

执行成功后，数据库中应有 9 张表：
`customers`、`purchase_records`、`rental_records`、`return_records`、
`sales_records`、`users`、`vcd`、`vcd_categories`、`vcd_inventory`

## 三、前端构建步骤

### 1. 进入前端目录
```
cd vcd-frontend
```

### 2. 安装依赖
```
npm install
```

### 3. 构建前端（打包到后端 resources/dist 目录）
```
npm run build
```

构建完成后，`vcd-backend/src/main/resources/dist/` 目录下会生成：
- `login.html`
- `manage.html`
- `assets/` 目录（JS、CSS、图片）

---

## 四、后端启动步骤

### 方法一：IDEA 直接运行（推荐）

1. 用 IDEA 打开 `vcd-backend` 文件夹
2. 等待 Maven 自动下载依赖（需要网络）
3. 找到 `BackendApplication.java`，右键 → Run
4. 看到 `Started BackendApplication` 即启动成功

### 方法二：Maven 命令行打包运行

```bash
# 进入后端目录
cd vcd-backend

# 打包（跳过测试）
mvn clean package -DskipTests

# 运行 jar 包
java -jar target/backend-0.0.1-SNAPSHOT.jar
```

## 五、访问系统

启动成功后，打开浏览器访问：

```
http://localhost:8080/login.html
```

**登录账号：**

首次使用需注册账号，启动后端后执行：

```
POST http://localhost:8080/api/users/register
Body: {"username":"admin","password":"123456"}
```

然后用 `admin` / `123456` 登录。

## 七、常见问题排查

### 问题1：端口 8080 被占用
```bash
# 查找占用端口的进程
netstat -ano | findstr :8080
# 结束对应进程（替换 PID）
taskkill /PID <PID> /F
```

### 问题2：数据库连接失败
检查 `application.properties` 中的配置：
```properties
spring.datasource.url=jdbc:sqlserver://localhost:1433;databaseName=vcddb;encrypt=false;trustServerCertificate=true
spring.datasource.username=sa
spring.datasource.password=123456
```
- 确认 SQL Server 服务已启动
- 确认 sa 账户已启用且密码正确
- 确认 TCP/IP 协议已启用

### 问题3：Maven 依赖下载失败
- 检查网络连接
- 可配置国内镜像源（阿里云）：在 Maven 的 `settings.xml` 中添加：
```xml
<mirror>
  <id>aliyun</id>
  <mirrorOf>central</mirrorOf>
  <name>阿里云公共仓库</name>
  <url>https://maven.aliyun.com/repository/public</url>
</mirror>
```

### 问题4：前端页面空白
- 确认已执行 `npm run build` 且构建成功
- 确认 `vcd-backend/src/main/resources/dist/` 目录下有 `login.html` 和 `manage.html`
- 重新启动后端

### 问题5：登录后跳转失败
- 检查浏览器控制台是否有报错
- 确认 `localStorage` 中有 `jwt_token`
- 确认后端正常返回 token

### 问题6：操作提示"VCD库存不存在"
- 新增VCD后，需要先在「库存管理」中为该VCD创建库存记录，才能进行采购/租赁/销售操作

## 九、Docker 部署（可选）

本项目支持两种 Docker 部署方式，详细步骤见 `docs/docker-guide.md`。

### 方式一：Docker 仅数据库容器化

**环境要求：** JDK 17 + Docker Desktop

```bash
# 启动 SQL Server 容器
docker run -d --name vcd-sqlserver ^
  -e "ACCEPT_EULA=Y" -e "SA_PASSWORD=123456" ^
  -p 1433:1433 ^
  mcr.microsoft.com/mssql/server:2022-latest

# 用 SSMS 连接 localhost,1433 执行 script.sql

# 运行 jar 包
cd deploy
java -jar backend-0.0.1-SNAPSHOT.jar
```

### 方式二：Docker Compose 完整容器化

**环境要求：** 仅需 Docker Desktop（无需 JDK 和 SQL Server）

```bash
# 在项目根目录执行，一键启动
docker-compose up -d --build

# 查看状态
docker-compose ps

# 停止服务
docker-compose down
```

访问：`http://localhost:8080/login.html`

> 注意：Docker Compose 方式的 sa 密码为 `123456Aa!`（SQL Server 容器密码规则要求含大小写和数字）
