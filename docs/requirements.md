# 需求文档（Requirements Document）

## 项目名称

音响店 VCD 零售出租管理系统

## 版本

v1.0.0

## 编写日期

2026-04-19

---

## 1. 项目背景与目标

### 1.1 背景

本系统为数据库课程设计项目，面向音响店日常经营场景，解决 VCD 碟片的进货、出租、销售、归还等业务的手工记录效率低、数据易丢失、统计困难等问题。

### 1.2 目标

- 实现 VCD 碟片信息的数字化管理
- 实现客户信息、租赁/销售/采购/归还记录的完整闭环管理
- 提供首页数据看板，直观展示经营状况
- 通过 JWT 认证保障系统安全访问

## 3. 功能需求

### 3.1 用户认证模块

| 编号 | 功能 | 描述 |
|------|------|------|
| F-01 | 用户登录 | 输入用户名和密码，验证通过后返回 JWT Token，跳转管理页 |
| F-02 | 用户注册 | 填写用户名和密码，密码经 BCrypt 加密后存储 |
| F-03 | 登录态保持 | Token 存储于 localStorage，有效期 24 小时 |
| F-04 | Token 过期处理 | 请求时检测 Token 是否过期，过期自动跳转登录页 |
| F-05 | 退出登录 | 清除 localStorage 中的 Token，跳转登录页 |
| F-06 | 用户管理 | 查看所有用户、修改密码、删除用户 |

### 3.2 首页数据看板模块

| F-12 | 待归还列表 | 展示当前未归还的租赁记录（含逾期标记） |

## 4. 非功能需求

| 编号 | 类型 | 描述 |
|------|------|------|
| NF-01 | 安全性 | 除登录/注册接口外，所有 API 均需携带有效 JWT Token |
| NF-02 | 安全性 | 用户密码使用 BCrypt 算法加密存储，不可逆 |
| NF-03 | 性能 | 列表接口响应时间 < 2 秒（本地环境） |
| NF-04 | 可用性 | 前端页面在 Chrome/Edge 最新版本下正常运行 |
| NF-05 | 数据完整性 | 数据库层面通过外键约束保证关联数据一致性 |
| NF-06 | 事务性 | 涉及库存变更的操作（采购/销售/租赁/归还）使用 @Transactional 保证原子性 |
| NF-07 | 编码 | 全系统统一使用 UTF-8 编码 |

## 5. 数据库表需求

系统共需 9 张数据库表：

| 表名 | 业务含义 |
|------|---------|
| `users` | 系统用户表 |
| `vcd_categories` | VCD 分类表 |
| `vcd` | VCD 信息表 |
| `vcd_inventory` | VCD 库存表 |
| `customers` | 客户信息表 |
| `purchase_records` | 采购记录表 |
| `rental_records` | 租赁记录表 |
| `return_records` | 归还记录表 |
| `sales_records` | 销售记录表 |

---

## 6. 约束与限制

- 数据库使用 **SQL Server 2019/2022**，端口 1433
- 后端运行环境：**JDK 17**，端口 8080
- 前端以静态文件形式内嵌于后端 jar 包，通过 Spring Boot 托管
- 不支持多用户并发权限控制（无角色区分）
- 不支持文件上传功能
